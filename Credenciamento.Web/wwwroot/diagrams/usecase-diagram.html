📚 Documentação dos Casos de Uso

👤 Visitante (Não Autenticado)

  • UC1: Visualizar eventos disponíveis na página inicial
  • UC2: Pesquisar eventos por data, nome ou local
  • UC4: Realizar cadastro no sistema

👤 Participante (Autenticado)

  • UC5: Fazer login no sistema
  • UC6: Atualizar dados do perfil
  • UC7: Comprar ingresso para evento
  • UC9: Visualizar ingresso adquirido
  • UC10: Imprimir ingresso com QR Code
  • UC12: Consultar histórico de ingressos

👤 Administrador

  • UC3: Gerenciar eventos (criar, editar, excluir)
  • UC11: Validar QR Code na entrada do evento

🏦 Sistema de Pagamento (Ator Externo)

  • UC8: Processar pagamento do ingresso
  • Integração com gateway de pagamento
  • Validação de cartão de crédito

🔗 Relacionamentos Include

  • UC7 → UC8: Comprar ingresso inclui processar pagamento
  • UC7 → UC4: Comprar ingresso inclui cadastro (se novo)
  • UC9 → UC5: Visualizar ingresso inclui autenticação
  • UC12 → UC5: Consultar histórico inclui autenticação

🔀 Relacionamentos Extend

  • UC10 → UC9: Imprimir ingresso estende visualizar ingresso
  • Funcionalidade opcional
  • Executada sob demanda do usuário

📋 Fluxo Principal - Comprar Ingresso

  • 1. Participante visualiza eventos
  • 2. Participante seleciona evento
  • 3. Sistema verifica autenticação
  • 4. Participante confirma compra
  • 5. Sistema processa pagamento
  • 6. Sistema gera ingresso com QR Code
  • 7. Sistema envia ingresso por e-mail

🎫 Pós-Condições

  • Ingresso armazenado no banco de dados
  • QR Code único gerado
  • E-mail de confirmação enviado
  • Pagamento confirmado
  • Status do ticket: Ativo
